# Case-Study Schema Upgrade Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add structured client-info fields to case-study articles, surface them in the admin editor, and render them on the public `/case-studies/[slug]` page.

**Architecture:** Five new optional columns on `Article` (`clientName`, `clientLocation`, `clientUrl`, `aiModels`, `mainChallenges`). A new reusable `<CaseStudyMetaFields>` component drives editing. Public page renders a "Company info" panel above the article body when any field is set. All fields nullable / defaulted so existing articles are unaffected.

**Tech Stack:** Prisma (PostgreSQL), Next.js 16 App Router, React 19, TypeScript, Tailwind v4, next-intl.

```tsx
        {m.aiModels.length ? (
          <div className="flex flex-col">
            <dt className="text-xs text-slate-500">{t('aiModels')}</dt>
            <dd className="flex flex-wrap gap-1.5">
              {m.aiModels.map((model) => (
                <span
                  key={model}
                  className="inline-flex items-center rounded-full bg-white border border-slate-200 px-2.5 py-0.5 text-xs text-slate-700"
                >
                  {model}
                </span>
              ))}
            </dd>
          </div>
        ) : null}
        {m.mainChallenges ? (
          <div className="md:col-span-2 flex flex-col">
            <dt className="text-xs text-slate-500">{t('mainChallenges')}</dt>
            <dd className="text-slate-700 whitespace-pre-line">
              {m.mainChallenges}
            </dd>
          </div>
        ) : null}
      </dl>
    </div>
  );
}
```

## Notes for the implementer

- **Snake_case map names matter:** the `@map("client_name")` etc. annotations are not optional — they keep the SQL column names consistent with the rest of `articles` table.
- **`aiModels` defaults to `[]`:** the type is `String[]`, never `String[]?` — Postgres arrays default to empty for new rows, which is what we want. The TS type on `CaseStudyMeta.aiModels` is also non-optional `string[]`.
- **`hasCaseStudyMeta` is exported from `@/types`:** if you put it elsewhere by accident, the public panel will need a different import path.
- **The conditional render in `PostMetaForm`** is gated by `category === 'case-study'`. If category changes mid-edit, the section appears/disappears but state is preserved in `PostEditor`, so toggling back doesn't lose data.
- **`buildArticlePayload` only emits `caseStudy` when `category === 'case-study'`** — this means switching away from case-study and saving will clear the fields server-side. That's intentional: a "Useful Info" article should not have stale client metadata.
- **Migration filename includes a timestamp prefix** like `20260623_...`. Prisma generates this automatically.
